Don't forget to save the plugin after configuring the languages.

Many languages are preconfigured here as examples but are deactivated. You can remove languages (red button) or add (green) or sort (blue).

Hyphenopoly checks with the here configured AND ACTIVATED languages if the browser supports hyphenation without additional tools. Only if not then Hyphenopoly tries to load a so called "pattern" for each unsupported language. These are files with file extension *.wasm, e.g. en-gb.wasm. You can see all available language patterns in the dropdown lists of column "Language (Hyphenopoly)".

Normally a Joomla page has at least 1 lang attribute, namely inside the html tag, e.g. in a german page <html lang="de-de" dir="ltr">.

If you run a multilingual Joomla site with a language switcher German/English there are also english pages with <html lang="en-gb" dir="ltr">.

That means: You must configure both languages in the plugin because they have different hyphenation rules.

Let's start with the easier one, english:

  • The lang attribute is en-gb and there's a pattern en-gb in the dropdown list in column Language (Hyphenopoly). Perfect! Select it and leave the field of column "lang Attribute" empty.
  • Enter a single and at least 12 characters long and polysyllabic word in column Word . For english it could be "hyphenationalgorithm".
  • Don't forget to activate the language in column "Active".

Let's now configure German that needs a so called "fallback" configuration:

  • The lang attribute inside page HTML is "de-de" and, bad news, there is NO pattern de-de yet (while I write this description in 2019) in the dropdown list in column "Language (Hyphenopoly)".
  • Try to find another similiar pattern and select it. I propose de.
  • In column "lang Attribute" enter "de-de". Thus Hyphenopoly knows that it shall fall back to pattern "de" if it finds a lang=de-de in the page.
  • Enter a single and at least 12 characters long and polysyllabic word in column Word . For german it could be "Silbentrennungsalgorithmus".
  • Don't forget to activate the language in column "Active".

The tabulator "Cleanup & Debug & Log" contains some infos about log messages that may help you to adjust your language settings.

Keep in mind:

Any here configured and activated language will be checked by Hyphenopoly. You should not activate unnecessary languages. Any unnecessary check eats up some performance of your site.

Experts:

The Word "FORCEHYPHENOPOLY" will force Hyphenopoly to load language patterns even if a browser like FireFox doesn't need them. And Hyphenopoly will use its hyphenation scripts. Note: Disable CSS-hyphenation ("hyphens" rules) while using "FORCEHYPHENOPOLY" for testing purposes.
