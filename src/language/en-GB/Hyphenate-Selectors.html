The CSS selectors that you enter in this field line-by-line will get a CSS class "hyphenate". The content of the marked HTML element and it's child elements will be hyphenated. The markup is the same like in a CSS file. You can enter HTML tags or CSS classes (with a leading dot) or ids (with a leading #) or nearly any other usual combinations.

Example with tag body

The visible content of all HTML elements inside the page will be hyphenated because the superordinate <body>-Tag gets a class "hyphenate" (<body class="hyphenate">) and all child elements of a so tagged element will inherit the behavior. Normally you must then exclude many of the child elements with the setting "Do-Not-Hyphenate-Selectors" to disable the hyphenation for these.

It's not recommended to configure "body" if you want to avoid performance issues. This means that you should only use the body tag for initial tests.

Define smaller containers and more specific. In Joomla that's normally only the output of components like an article.

Tipp for the lazy ones:

  • Surround <jdoc:include type="component" /> inside your template-index.php with an additional "div" container.. 
    <div class="makeMeHyphenated">
<jdoc:include type="component" />
</div>
  • Then enter .makeMeHyphenated in this field.
  • You can also write <div class="hyphenate"> and enter a "nonsense selector" in this field (see below why) but enter never .hyphenate in this field!
Example with CSS class .item-page

All tags (elements) with a class item-page will get automatically an additional class hyphenate. <div class="item-page something-else"> => <div class="item-page something-else hyphenate">

Example with CSS id #icons

A <div id="icons" class="small some-other-class"> will become <div id="icons" class="small some-other-class hyphenate">

Enter 1 selector per line.

Some more examples:

.item-page
.item p
#icons.small
div.container
section.somewhere div.inside
#myid .outside
[itemprop="headline"]
[itemprop="articleBody"]

If you leave this field empty and field "Do-Not-Hyphenate-Selectors" is empty, too, the plugin will NOT load the hyphenation scripts. Enter a "nonsense selector" like "nonsensexyzzz" if you want to force loading of the scripts, e.g. if you prefer to insert the classes "hyphenate" and "donthyphenate" manually in your HTML markup.
