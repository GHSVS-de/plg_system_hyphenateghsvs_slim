Regardless of the Hyphenopoly console messages mentioned above, you can enable additional log messages that will be written to a file if you select "YES" here.

Ultimately, this feature does not provide much more information than the Hyphenopoly console messages. But when combined with the "Cleanup" setting, it also logs page links, which may be helpful.

You do not have to look into the browser error console individually for each page, but can collect messages across multiple pages. Also over several days. Activate the feature and click in the frontend a few pages and languages (if you use a multilingual page). Look here then, what was collected.

You can find out how the file is called and where it is stored and its file size by clicking on the "Show Log File Path and Size" button. You can download there the file, too.

Be aware that showing of very large files with button "Show Log File Content" can take ages. Be patient after clicking on the button! You should use the download feature above.

Don't forget to delete it before it gets too big after you've doenloaded ist. Since the "Log" feature tries to log every message only once, oversized files can slow the system down.

Deactivate "Log" if you're debug work is done.
