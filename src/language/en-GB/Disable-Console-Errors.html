In two cases Hyphenopoly shows informatory messages inside the console of your browser:

  1. Hyphenopoly finds a "lang" attribute within the page (e.g. lang=en-gb) that is not configured in the plugin (= needed language pattern not loaded) and if the found attribute is in an area that shall be hyphenated (by setting Hyphenate-Selectors).
  2. Hyphenopoly finds a language in the plugin configuration that is not needed in the page (= unneccessary language pattern loaded).

Examples:

  1. Element with 'en-gb' found, but 'en-gb.wasm' not loaded. Check language tags!
  2. engine for language 'fr' loaded, but no elements found.

"Normal" visitors of your page won't see the messages and Hyphonopoly will not stop work but perhaps doesn't hyphenate correctly.

These messages can be very helpful in correcting your plugin configuration.

Messages of type 1. tell you that you haven't configured a needed language for a lang tag en-GB. Solution: Configure it correctly in the plugin.

Type 2 messages will appear on almost any Joomla page if you run a multilingual site, even if you have properly configured the plugin for all languages. It is clear that in most cases a German page does not contain English elements labeled lang=en-gb. And vice versa.

If you no longer need these Hyphenopoly messages in the browser console, set this to "YES."
