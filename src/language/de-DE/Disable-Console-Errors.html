In zwei Fällen zeigt Hyphenopoly in der Konsole Ihres Browsers informative Meldungen an:

  1. Hyphenopoly findet ein "lang"-Attribut innerhalb der Seite (z.B. lang=en-gb), das im Plugin nicht konfiguriert ist und wenn sich das gefundene Attribut in einem Bereich befindet, der getrennt werden soll (= nötiges Sprach-Pattern nicht geladen).
  2. Hyphenopoly findet in der Plugin-Konfiguration eine Sprache, die innerhalb der Seite nicht gebraucht wird (= unnötiges Sprach-Pattern geladen).

Beispiele:

  1. Element with 'en-gb' found, but 'en-gb.wasm' not loaded. Check language tags!
    = 'en-gb' gefunden, aber Pattern 'en-gb.wasm' nicht geladen.
  2. engine for language 'fr' loaded, but no elements found.
    = Pattern 'fr.wasm' geladen, aber wird gar nicht gebraucht.

"Normale" Besucher Ihrer Seite sehen diese Nachrichten zwar nicht und Hyphonopoly stoppt die Arbeit nicht, aber trennt möglicherweise nicht richtig.

Diese Meldungen können sehr hilfreich sein, um Ihre Plugin-Konfiguration zu korrigieren.

Mitteilungen vom Typ 1. teilen Ihnen mit, dass Sie keine Sprache für ein gefundenes "lang"-Attribut "en-GB" konfiguriert haben. Lösung: Konfigurieren Sie die Sprache korrekt im Plugin.

Nachrichten vom Typ 2 werden fast auf jeder Joomla-Seite angezeigt, wenn Sie eine mehrsprachige Site betreiben, auch wenn Sie das Plugin für alle Sprachen richtig konfiguriert haben. Es ist klar, dass eine deutsche Seite in den meisten Fällen keine englischen Elemente enthält, die mit lang=en-gb gekennzeichnet sind. Und umgekehrt.

Wenn Sie diese Hyphenopoly-Nachrichten in der Browser-Konsole nicht mehr benötigen, stellen Sie hier "JA" ein.
